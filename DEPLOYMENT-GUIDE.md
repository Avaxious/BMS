 // 90 روز\n  \n  let rowsDeleted = 0;\n  for (let i = data.length - 1; i > 0; i--) {\n    const logDate = new Date(data[i][0]);\n    if (logDate < cutoff) {\n      sheet.deleteRow(i + 1);\n      rowsDeleted++;\n    }\n  }\n  \n  logSecurityEvent('CLEANUP_COMPLETED', 'SYSTEM', `Deleted ${rowsDeleted} old logs`);\n}\n```\n\n---\n\n## مرحله ۱۰: Production Checklist\n\n**قبل از استقرار نهایی، تمام موارد را چک کنید:**\n\n```\n☐ AuditLog sheet ایجاد شد\n☐ Users sheet headers صحیح هستند\n☐ Security-Config.gs در Login Proxy موجود است\n☐ Login Proxy Enhanced.gs deploy شد\n☐ Security-Config.gs در Data Proxy موجود است\n☐ Data Proxy Enhanced.gs deploy شد\n☐ index-security-enhanced.html URLs update شدند\n☐ Frontend در GitHub Pages نمایش داده می‌شود\n☐ Login test موفق بود\n☐ Rate limiting test موفق بود\n☐ Audit logs ثبت شدند\n☐ Password hashing کار می‌کند\n☐ HTTPS فعال است\n☐ Backup automation تنظیم شد\n```\n\n✅ **اگر تمام موارد ✓ شدند، سیستم آماده برای Production است!**\n\n---\n\n## مرحله ۱۱: اضطراری Response\n\n### اگر نقص امنیتی شناسایی شود:\n\n1. **فوری:**\n   ```\n   ☐ تمام tokens را غیرفعال کنید\n   ☐ Column D (Token) تمام صفحات را خالی کنید\n   ☐ Audit logs بررسی کنید\n   ```\n\n2. **در 1 ساعت:**\n   ```\n   ☐ Passwords تمام کاربران تغییر کنید\n   ☐ AuditLog تحلیل کنید\n   ☐ Data Sheets چک کنید (نشت داده)\n   ```\n\n3. **در 24 ساعت:**\n   ```\n   ☐ نتیجه گزارش بنویسید\n   ☐ به کاربران اطلاع دهید\n   ☐ Patches deploy کنید\n   ```\n\n---\n\n## پشتیبانی\n\n### اگر مشکلی پیدا کردید:\n\n1. **Logs بررسی کنید:**\n   - Apps Script Logs: `View` → `Logs`\n   - AuditLog Sheet: Events را ببینید\n\n2. **Debugging:**\n   ```javascript\n   // Temporary logging اضافه کنید:\n   Logger.log('Variable value: ' + JSON.stringify(variable));\n   ```\n\n3. **GitHub Issues:**\n   - Issues tab را باز کنید\n   - Issue جدید ایجاد کنید\n   - Steps to reproduce را بنویسید\n\n---\n\n**نسخه:** 1.0  \n**آپدیت:** 2026-07-19  \n**Status:** ✅ Ready for Production\n"
